Git Workflow:

  1. Get the latest version of master
  2. Create your own feature branch, prefixed by your initials. Branches should ideally be scoped to one feature, and short-lived.
  3. git checkout -b xz-feature
  4. Discuss the feature with another developer, they should eventually review your code.
  5. Do some work
  6. Push to a remote branch, rebasing beforehand if you have redundant commits which should be squashed
  7. Create a Pull Request
  8. Ask another developer to review the code
  9. Reply to comments and make amends, if necessary
  10. Merge into master once you receive a +1
  11. Delete your feature branch
