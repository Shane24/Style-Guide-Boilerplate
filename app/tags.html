Component Tags

Component tags can be used to identify whether components are being actively developed, dev-ready and stable, or deprecated, etc.

Tags could also be used as part of the library build system, eg: Components marked with 'deprecated' tags could excluded from production builds, but remain in the library for future reference or legacy implementations.

Other tags could be introduced if necessary. These are just examples.

The tags are:

In-Progress

A component with this tag is under development, and is not ready for production use.


Deprecated

A component with this tag is no longer to be used. The component may have been replaced by a better or alternative component, etc.


Stable

A component with this tag is ready for use in production applications.
