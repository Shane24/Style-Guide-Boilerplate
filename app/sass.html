Rules

Avoid element selectors

Use child or descendant selectors with element selectors if the element selectors will and can be predictable. Using .module span is great if a span will predictably be used and styled the same way every time while within that module.

However, as a project grows in complexity, the more likely that you will need to expand a component’s functionality and the more limited you will be in having used such a generic element within your rule.

Do not use id's for styling

IDs may be used as fragment identifiers, and for JavaScript’s getElementById etc. They should never be used to style an element.

Sass Inception Rule

Sass nesting should never be more than 3 levels deep. If you can limit it to 1 level deep, even better! We want highly performant CSS.

Be cautious using @extend

There are multiple reasons not to use @extend. However it can be used without making a mess with the help of Sass's %placeholder selectors. Only use the @extend directive to extend placeholders, not other selectors.

Comments

Write code to be readable and understandable, not to save bytes. Code is written and maintained by people. Ensure your code is descriptive, well commented, and approachable by others. Great code comments convey context or purpose. Do not simply reiterate a component or class name.

Be sure to write in complete sentences for larger comments and succinct phrases for general notes.

Javascript hooks

Classes used specifically for JS hooks should be given the prefix "js-". This ensures that we maintain a distinction between content and functionality. These classes should never be used for style.

Variable names

Generic, modular names for variables should be used.

If specificity is needed, variables should still be Generic > Specific in their naming convention.

This not only helps in recollection, but will allow a text editor (such as Sublime Text, Coda etc.) to easily suggest colors. This way you don't have to memorize exactly how you named your variables. Rather, you can start generically and get more specific as the text editor auto-suggests variable names.
